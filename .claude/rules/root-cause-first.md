# Сначала причина, потом фикс

Не лепи фиксы на симптомы. Если что-то сломалось — сначала пойми **почему**, потом чини.

## Порядок действий

1. Собери факты: что работало, когда перестало, что изменилось
2. Найди противоречия между фактами и проверь каждое
3. Не предлагай workaround пока не объяснишь, почему раньше обходились без него
4. Только когда причина найдена (или доказано что найти невозможно) — пиши фикс
